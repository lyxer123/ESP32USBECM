# USB ECM 双向拓扑切换 — 功能与调试说明

**文档日期：** 2026年3月28日  
**修订说明：** 补充工程修改清单、实施过程、关键源码位置及已踩坑与对策；与当前仓库 `main/` 下实现一致。

**ESP-IDF 版本：**  
- **5.2.x**、**5.5.x**：可正常完成工程配置与编译。切换大版本后请执行 **`idf.py reconfigure`**（或删除过期的 **`dependencies.lock`** 再配置），使锁文件里的 **`cmd_system`** 等路径指向当前 **`IDF_PATH`**；若锁文件仍指向另一套 IDF（例如 5.5 的示例树在 5.2 上编译），可能出现 **无法解析 `esp_driver_uart`** 等 CMake 错误。  
- **5.1.x**：**不可用**。依赖链中的 **`iot_usbh_ecm`** 在 `idf_component.yml` 中声明 **`idf: ">=5.2"`**，低于 5.2 时无法满足依赖，**无法按本工程通过编译**。

**功能概述：** 在 CH397A（USB ECM）与 ESP32（如 ESP32-S3）组合使用时，通过 `menuconfig` 中的 **USB ECM network topology** 在两种数据路径间切换，无需改应用逻辑：  
- **模式 A**：上游经 ECM（DHCP）、下游 Wi‑Fi AP + NAPT；  
- **模式 B**：上游 Wi‑Fi STA（DHCP）、下游 ECM 静态网关 + DHCPS + NAPT 给 RJ45 侧 PC。  

两种模式编译期二选一；默认保持模式 A，与原有示例行为兼容。

---

## 一、实现了什么功能

### 1. 模式 A：Router/PC — [CH397 ECM] — ESP32 — Wi‑Fi AP（默认）

- RJ45 接路由器或 PC 共享网络等，ESP 在 USB ECM 上 **DHCP 客户端** 获取上游 IP。  
- 随后 **SoftAP**，终端经 **NAPT** 走 ECM 出口。  
- **默认网卡**：`s_ecm_netif`（ECM）。

**CH397 转 Wi‑Fi（模式 A）时，接入热点一侧的 IP 网段**

- SoftAP 使用与 **`menuconfig` → `4G Modem WiFi Config` → `AP 网关地址`（`CONFIG_SERVER_IP`）** 一致的网段；工程 **默认**为 **`192.168.4.1/24`**。手机或 PC 作为 **STA 连热点** 时，地址应在 **`192.168.4.x`**（与 ECM 上游网段无关）。  
- **推荐**：终端上 **IPv4 设为自动获取（DHCP）**，由 ESP 分配 `192.168.4.x`，无需手填。  
- **若必须手工指定静态 IP**（部分 PC「共享」或企业策略会干扰 DHCP）：  
  - **IP 地址**：`192.168.4.2`～`192.168.4.254` 中选未被占用的一个（勿与网关冲突）；  
  - **子网掩码**：`255.255.255.0`；  
  - **默认网关** 与 **DNS**：均填 **`192.168.4.1`**（若你改过 `CONFIG_SERVER_IP`，则改为与该项 **同一网段的网关地址**，例如网关为 `10.0.0.1` 时，终端应使用 `10.0.0.x`）。  
- **常见误区**：不要把 Wi‑Fi 客户端设成 RJ45/CH397 上游网段（如 `192.168.137.x`）；那是网线侧 DHCP 的网段，与热点子网是两层不同的网络。

### 2. 模式 B：Wi‑Fi STA — ESP32 — [CH397 ECM] — PC

- ESP **STA** 连现有 Wi‑Fi，DHCP 获上游地址。  
- ECM 侧 **静态 IPv4**（Kconfig 默认可为 `192.168.137.1/24`），对 PC 开 **DHCPS**，PC 网关指向该地址。  
- **`ip_napt_enable`** 在 ECM 侧地址上使能，PC 流量 **NAT 到 STA**。  
- **默认网卡**：STA，便于设备本机 HTTP/测速/ping 走 Wi‑Fi。  
- 反向选项在 Kconfig 中 **`select LWIP_IP_FORWARD`** 与 **`select LWIP_IPV4_NAPT`**。

## 二、如何配置（menuconfig）

1. `idf.py menuconfig` → **Example Configuration → USB ECM network topology** 二选一。  
2. 模式 B 下额外项：**ECM LAN IP**、**ECM LAN netmask**。  
3. **4G Modem WiFi Config** 中填写 **STA SSID / Password**（模式 B 必需）。  
4. 保存后 `idf.py build` / 烧录。

## 四、实施过程与迭代（问题 → 对策）

以下为实际开发与联调中的顺序，便于复现理解与排错。

1. **需求**  
   在 **不破坏**「ECM 上游 + Wi‑Fi AP」的前提下，增加「Wi‑Fi STA 上游 + ECM 给 PC」路径，并用 **sdkconfig / menuconfig** 切换。

2. **Kconfig**  
   增加 `EXAMPLE_NETWORK_TOPOLOGY` 二选一及 ECM 下游网段字符串项；反向项 `select` `LWIP_IP_FORWARD`、`LWIP_IPV4_NAPT`。

3. **`app_main` 顺序**  
   - 模式 A：`install_ecm` → 等 **ECM GOT_IP** → `app_wifi_main`(AP) → `esp_netif_set_default_netif(ecm)`。  
   - 模式 B：`install_ecm` → `app_wifi_main`（**强制 `WIFI_MODE_STA`**）→ 等 **STA GOT_IP** → 等 **ECM 链路 up** → `ecm_start_dhcps_and_napt` → `esp_netif_set_default_netif(sta)`。

4. **iperf 与反向模式**  
   原先把 `iperf_init()` 放在 `app_wifi_main` **之前** 且 `vTaskSuspend`，反向模式下 **永远不会** 起 STA/DHCPS。  
   **对策**：仅 **模式 A** 在「ECM 已拿 IP」后早退进 iperf；**模式 B** 在 **网络栈就绪后** 再 iperf。

5. **`ESP_ERR_ESP_NETIF_IF_NOT_READY`（`esp_netif_dhcps_option`）**  
   IDF 中若 **`esp_netif->dhcps == NULL`** 即返回该错。`dhcps` 仅在 **`esp_netif_new()` 时 `flags` 含 `ESP_NETIF_DHCP_SERVER`** 才会 `dhcps_new()`。  
   **对策**：在模式 B 下基于 **`ESP_NETIF_INHERENT_DEFAULT_ETH()`**，**去掉 `ESP_NETIF_DHCP_CLIENT`**，**再或上 `ESP_NETIF_IPV4_ONLY_FLAGS(ESP_NETIF_DHCP_SERVER) | ESP_NETIF_FLAG_AUTOUP`**，保留默认 ETH 其余标志位；`esp_netif_new` 后打日志校验 `DHCP_SERVER` 位。

6. **STA 早于 ECM 就绪**  
   `dhcps_option` 需在 ECM **链路已连接**、netif 就绪后调用更稳妥。  
   **对策**：增加 **`EVENT_ECM_LINK_UP_BIT`**，在 **`IOT_ETH_EVENT_CONNECTED`** 置位、**DISCONNECTED** 清除（仅模式 B）；在 STA `GOT_IP` 之后 **`WaitBits(EVENT_ECM_LINK_UP_BIT)`** 再启动 DHCPS。

7. **`ESP_ERR_ESP_NETIF_DHCP_NOT_STOPPED`（`esp_netif_set_ip_info`）**  
   对 **DHCP_SERVER** 类 netif，`set_ip_info` 要求 **`dhcps_status == ESP_NETIF_DHCP_STOPPED`**；新建后常为 **INIT**。  
   **对策**：在 **`esp_netif_set_ip_info` 之前** 调用 **`esp_netif_dhcps_stop`**（允许 `ESP_ERR_ESP_NETIF_DHCP_ALREADY_STOPPED`）。

8. **组件路径 / Git**  
   原 `../../../../../components/...` 在独立目录克隆时会解析到错误盘符路径。  
   **对策**：`main/idf_component.yml` 改为相对本仓库的 **`../../esp-iot-solution/...`**；根目录 **`.gitignore`** 排除 `build/`、`.cache/`、`.vscode/`。

## 五、关键源代码说明

以下路径均相对于仓库根目录；行号为撰写本文时的大致位置，若后续编辑有偏移，请以文件内实际内容为准。

### 5.1 Kconfig（`main/Kconfig.projbuild`）

- **`EXAMPLE_NETWORK_TOPOLOGY`**：`EXAMPLE_TOPO_ECM_ROUTER_WIFI_AP` / `EXAMPLE_TOPO_WIFI_STA_ECM_PC`。  
- **`EXAMPLE_ECM_DOWNSTREAM_GATEWAY_IP` / `EXAMPLE_ECM_DOWNSTREAM_NETMASK`**：`depends on EXAMPLE_TOPO_WIFI_STA_ECM_PC`。  
- **`EXAMPLE_TOPO_WIFI_STA_ECM_PC`**：`select LWIP_IP_FORWARD`、`select LWIP_IPV4_NAPT`。

### 5.2 事件位与 `iot_event_handle`（`main/usb_ecm_4g_module.c`）

- `EVENT_GOT_IP_BIT`、`EVENT_AT_READY_BIT`、**`EVENT_ECM_LINK_UP_BIT`**。  
- **模式 B**：`IOT_ETH_EVENT_CONNECTED` 置位 `EVENT_ECM_LINK_UP_BIT`；断开清除。  
- **模式 A**：`IOT_ETH_EVENT_DISCONNECTED` 仍清除 `EVENT_GOT_IP_BIT` 并停 ping；模式 B 不在此清除 STA 相关「已拿 IP」状态。  
- **IP 事件**：模式 A 注册 **`IP_EVENT_ETH_GOT_IP`**；模式 B 注册 **`IP_EVENT_STA_GOT_IP`**，并在回调里 **`esp_netif_set_default_netif(app_wifi_get_sta_netif())`**。

### 5.3 `install_ecm()`（同一文件）

要点摘要：

1. **`esp_netif_inherent_config_t`**：模式 B 下从默认 ETH 标志 **去掉 CLIENT、加上 SERVER|AUTOUP**，以便 **`dhcps_new`** 执行。  
2. **`esp_netif_new` 后**：日志打印 **`esp_netif_get_flags`**，无 **`ESP_NETIF_DHCP_SERVER`** 则 **`abort`** 并提示 fullclean 重建。  
3. **`esp_netif_attach` 后**（模式 B）：**`esp_netif_dhcps_stop`** → **`esp_netif_set_ip_info`**（静态 ECM 地址）→ **`iot_eth_start`**。

### 5.4 `ecm_start_dhcps_and_napt()`（仅模式 B 编译）

- 校验 flags 含 **`ESP_NETIF_DHCP_SERVER`**。  
- **`esp_netif_dhcps_stop`**（容忍 **`ESP_ERR_ESP_NETIF_DHCP_ALREADY_STOPPED`**）。  
- **`esp_netif_dhcps_option`**（`OFFER_DNS`）、**`esp_netif_set_dns_info`**（优先 STA 主 DNS，否则 8.8.8.8）、**`esp_netif_dhcps_start`**。  
- **`ip_napt_enable(ecm_ip.ip.addr, 1)`**。

### 5.5 `app_main()`（同一文件）

- 按拓扑分支 **`xEventGroupWaitBits`**、**`s_modem_wifi_config.mode = WIFI_MODE_STA`**、**等待 ECM 链路**、**`ecm_start_dhcps_and_napt`**、**`esp_netif_set_default_netif`**。  
- **iperf**：**`CONFIG_EXAMPLE_USB_ECM_IPERF && CONFIG_EXAMPLE_TOPO_ECM_ROUTER_WIFI_AP`** 与 **`... && CONFIG_EXAMPLE_TOPO_WIFI_STA_ECM_PC`** 分两处，保证反向模式先完成 Wi‑Fi + DHCPS。

### 5.6 头文件与依赖（`main/usb_ecm_4g_module.c` 顶部）

- 增加 **`string.h`**、**`dhcpserver/dhcpserver.h`**、**`lwip/ip_addr.h`**、**`lwip/lwip_napt.h`**，供 DHCPS/NAPT 与 `ipaddr_addr` 使用。

    _inherent_eth_config.if_key = netif_name;
    _inherent_eth_config.if_desc = netif_name;
#if CONFIG_EXAMPLE_TOPO_WIFI_STA_ECM_PC
    /* esp_netif_new() allocates dhcps only if ESP_NETIF_DHCP_SERVER is set (see esp_netif_lwip.c).
     * Start from default ETH flags, drop DHCP_CLIENT, add DHCP_SERVER|AUTOUP — same as SoftAP pattern. */
    {
        esp_netif_flags_t f = _inherent_eth_config.flags;
        f &= ~(esp_netif_flags_t)ESP_NETIF_DHCP_CLIENT;
        f |= (esp_netif_flags_t)(ESP_NETIF_IPV4_ONLY_FLAGS(ESP_NETIF_DHCP_SERVER) | ESP_NETIF_FLAG_AUTOUP);
        _inherent_eth_config.flags = f;
    }
#endif
    esp_netif_config_t netif_cfg = {
        .base = &_inherent_eth_config,
        .driver = NULL,
        .stack = ESP_NETIF_NETSTACK_DEFAULT_ETH,

    s_ecm_netif = esp_netif_new(&netif_cfg);
    if (s_ecm_netif == NULL) {
        ESP_LOGE(TAG, "Failed to create network interface");
        return;
    }
#if CONFIG_EXAMPLE_TOPO_WIFI_STA_ECM_PC
    {
        esp_netif_flags_t got = esp_netif_get_flags(s_ecm_netif);
        ESP_LOGI(TAG, "ECM netif flags=0x%x (DHCP_SERVER=%d)", (unsigned)got,
                 (got & ESP_NETIF_DHCP_SERVER) ? 1 : 0);
        if ((got & ESP_NETIF_DHCP_SERVER) == 0) {
            ESP_LOGE(TAG, "ECM netif has no DHCP_SERVER flag; dhcps handle was not created. idf.py fullclean && rebuild.");
            abort();
        }
    }

    glue = iot_eth_new_netif_glue(eth_handle);
    if (glue == NULL) {
        ESP_LOGE(TAG, "Failed to create netif glue");
        return;
    }
    esp_netif_attach(s_ecm_netif, glue);
#if CONFIG_EXAMPLE_TOPO_WIFI_STA_ECM_PC
    {
        /* esp_netif_set_ip_info() requires dhcps_status == STOPPED for DHCP_SERVER netifs; initial state is INIT. */
        esp_err_t derr = esp_netif_dhcps_stop(s_ecm_netif);
        if (derr != ESP_OK && derr != ESP_ERR_ESP_NETIF_DHCP_ALREADY_STOPPED) {

        ip_info.gw.addr = ip_info.ip.addr;
        ip_info.netmask.addr = ipaddr_addr(CONFIG_EXAMPLE_ECM_DOWNSTREAM_NETMASK);
        ESP_ERROR_CHECK(esp_netif_set_ip_info(s_ecm_netif, &ip_info));
    }

## 八、调试建议

- USB：CH397 多 configuration 时保持 **`CONFIG_USB_HOST_ENABLE_ENUM_FILTER_CALLBACK`** 等与原文档一致。  
- **模式 B 日志顺序**：STA **`WiFi STA GOT_IP`** → **`Waiting for USB ECM link up...`** → **`IOT_ETH_EVENT_CONNECTED`** → **`ECM downstream: DHCPS + NAPT`**。  
- PC 侧：应获得与 **ECM LAN IP** 同网段地址，网关为 **ECM LAN IP**；再测 DNS 与外网。  
- 异常时对照 **第四节** 中的错误码逐项排查；修改 Kconfig 后建议 **`idf.py fullclean`** 再编，避免旧目标文件残留。
